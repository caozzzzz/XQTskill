# 通用知识库 Markdown 格式

## 文档层级

- 一份 Markdown 文档对应一个领域、产品、制度、流程或知识集合。
- 第一个一级标题介绍领域整体信息。
- 后续一级标题分别对应主题、模块、章节、问题类型或流程分组。
- 主题下的二级标题可以包含 Mermaid 流程和知识内容。
- 三级标题 `### [条目名称] ###` 对应可检索知识条目。

## 条目字段

推荐使用通用字段：

    【所属领域】：领域名称
    【所属主题】：主题名称
    【所属条目】：条目名称
    【责任主体】：人员、系统、部门、对象或来源
    【知识内容】：约束、规则、解释、步骤、口径或事实

兼容旧字段：

    【所属场景】、【所属分支】、【所属节点】、【操作主体】、【业务规则】

`知识内容` 可以从字段下一行开始，并持续到下一个字段、三级标题或文档结尾。过滤空内容以及 `nan`、`none`、`null`。

## 流程字段

    ## 主题名称流程 ##
    【所属领域】：领域名称
    【所属主题】：主题名称
    【流程图】：
    ```mermaid
    flowchart LR
    ...
    ```

Mermaid `subgraph` 名称表示责任主体，节点方括号内容表示条目名称，箭头表示处理顺序。

## 主题概览字段

在文档开头的领域概览中提供主题列表：

    【领域名称】：领域名称
    【主题名称】：主题一、主题二、主题三

兼容旧字段：

    【场景名称】、【场景分支名称】、【分支名称】

## 检索优先级

1. 领域名称匹配文档。
2. 主题名称限定范围。
3. 条目名称精确匹配。
4. 查看条目知识内容。
5. 层级匹配失败后使用关键词或向量召回。
